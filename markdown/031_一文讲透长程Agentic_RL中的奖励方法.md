# 一文讲透：长程Agentic RL中的奖励方法

这不是对已有论文的总结或阐释，而是我最近一年以来在长程 Agentic RL 工作中的思考。

## 01 背景与定义

长程（Long-Horizon）Agentic RL 指 50+ 轮的 agentic 对话，模型需要利用工具调用来完成以下流程：

问题理解

环境探索

重现问题

尝试解决方案

再次重现问题以验证是否解决

典型场景是编程。

核心痛点：

训练提升不明显

容易出现熵坍塌（entropy collapse）

行为退化：模型一开始还会努力复现问题、验证解决方案，但随着训练步数增加，最后只会盲目修补（blind fix）——在某些问题上达到 100% 通过率，其它问题上退化到 0%

训练效率低

异步训练下，因 rollout 时间长导致离策略度（off-policiness）严重

其中，训练效率和 off-policy 问题属于基础设施层面的挑战，不在本文讨论范围内。

本文聚焦于行为退化这一核心问题，从奖励建模的角度分析其成因并探讨解决方案。

## 02 MDP 与 POMDP

2.1 基本定义

MDP（Markov Decision Process，马尔可夫决策过程） 是标准 RL 的基本框架，由五元组定义：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_002.png)

在 MDP 中，智能体始终能够观测到完整的状态 s，即信息是完备的。

POMDP（Partially Observable MDP，部分可观测马尔可夫决策过程）在 MDP 的基础上增加了观测的不完备性：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_003.png)

2.2 为什么这很重要

已有算法代表 GRPO 主要适用于 reasoning 场景，属于 MDP，但 Agentic RL 属于 POMDP。这一区别是后续所有分析的基础。

在 reasoning 场景中，问题和已生成的推理步骤构成了完整的“状态”——所有必要信息都在上下文里，不存在隐藏信息。

在一个典型的数学推理的过程中，模型 prefill 完整的 prompt 之后，其内部状态就已经足够解码出之后的所有证明过程。

也正因如此，采样序列里的每个 token 都是采样前的中间状态的延续，它们共同表征了模型在采样发生前的单一“信念”，所以 GRPO 将奖励平均分给一条样本里的所有 token 在这里是合理的。RL 需要解决的是解码过程中一次次分叉路径的选择问题。

补充：其实有些近期的 credit assignment 方向的研究也在隐隐约约地反驳这个观点。

那些研究认为其中的 pivot token，比如“然而”、“等一下”，之类的转折 token 会翻转之前的“信念”，使一个采样序列里出现不同的“信念”，从而需要不同的价值分配。

我认为采样过程本身也算是一种对环境的交互：本来 50% 概率发生的事情经过采样，对于后续 token 来说就是 100% 发生的事实了，这个变化确实会导致“信念”的变化。在这里不作深究。

而在 agentic 场景中，大量信息隐藏在环境里（文件系统、测试输出、运行时错误等），智能体需要主动探索才能获取。

每一次 tool call 都是和环境的一次交互，并且会带来更多的信息。仅仅靠 prefill 完整 prompt 之后的内部状态是无法直接给出最终答案或推理出关键的解决问题的步骤的。

这个过程相比于 math reasoning，其更类似于 car pole 这种传统的 RL 问题。

每次这样的交互都会带来更多的信息，从而大幅干扰“信念”的分布。GRPO 将奖励平均分配这些拥有着不同“信念”步骤直觉上是非最优的。

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_004.png)

通过消除不可见信息（这个过程就是探索——阅读文件、grep、agentic search 等），会减少奖励分布的不确定性：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_005.png)

RLVR（Reinforcement Learning with Verifiable Rewards）给出的可验证奖励本身是从一个高方差分布中采样的结果，直接用它来训练策略会引入大量噪声。

由于 rollout 代价高昂，同一个 prompt 的一次 rollout 总数很少（比如 8），其中步骤又非常多，奖励的采样平均与实际 r分布的偏差会很大，这进一步加剧了训练的困难。

既然真实奖励有分布、有采样，有期望，那就可以训练模型预测该期望和分布。

## 04 GRPO 的局限性

GRPO 给所有步骤的所有 token 设置统一的分数。

对于 MDP 来说，这背后蕴含着一个简单的思维：在 MDP 中，每一步对最终结果的贡献可以被均匀地归因——因为每一步都是在同一个信息基础上进行推理，推理链条中每一步的重要性大致相当。

但在 agentic POMDP 中并非如此。在智能体环境里，每次 tool call 都是一次和环境的交互，tool call 的 response 是难以预测的。获得 tool call response 可能会完全推翻之前的假说。

这个逻辑是顺畅的：并不是说最开始的假说就是错误的——假说的合理性并不能完全通过后验知识来判断，而是只能基于当时的有限信息。

所以很多基于“假说” → “归因” → “结果检验” → “奖惩假说”的方法在智能体环境中都不奏效。我们需要一种办法给不同步骤的合理性以不同的分数。

此外，GRPO 的均匀归因在 agentic 场景中还会引发一个更严重的问题——熵坍塌。这一点在下一章详细讨论。

## 05 熵坍塌：原因与对策

5.1 RL 的本质：用熵换性能

最近的研究（The Entropy Mechanism of Reinforcement Learning for Reasoning Language Models）揭示了一个重要的经验规律：RL 的本质是以策略熵换取性能。

在大量 RL 训练运行中，如果不对熵进行干预，策略熵会在训练早期急剧下降，导致策略过度自信，探索能力丧失，最终性能饱和。

该工作给出了一个简洁的经验公式：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_009.png)

从机制上看，策略熵的变化由动作概率与 logits 变化的协方差驱动。在使用 Policy Gradient 类算法时，这一协方差与优势函数成正比：高概率动作配合高优势 → 该动作概率进一步升高 → 熵下降。

实证表明这个协方差项在训练中大部分时间为正，这解释了为什么策略熵会单调递减。

5.2 Agentic RL 中熵坍塌的特殊成因

在 agentic RL 中，熵坍塌还有一个独特的加速因素，且与行为退化高度相关。

在长程 agentic 轨迹中，很大一部分步骤是探索行为——阅读文件、搜索代码、运行诊断命令等。

这些探索行为有两个特点：

本质上是随机的：模型不知道哪个文件包含关键信息，只能广撒网式地搜索。在概率分布上，这些探索步骤天然具有高熵。

难以归因到最终奖励：一次 grep 的输出可能只是排除了一个可能性（没有直接产出），但它对最终成功是不可或缺的。

在 GRPO 的均匀归因框架下，这些高熵的探索步骤得到的平均奖励很低——因为 GRPO 把所有步骤视为同等贡献，而探索步骤不直接产生奖励。

于是：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_010.png)

一旦策略不再探索，模型就无法获取解决困难问题所需的隐藏信息。它只能走“盲修”（blind fix）的捷径——不读代码、不复现问题，直接基于表面症状猜测一个修复。

这在简单问题上可能碰巧有效（因为简单问题的修复模式在训练数据中频繁出现），但在需要深入理解的困难问题上完全失败。这就是行为退化的机制：某些问题达到 100% 通过率，其它问题退化到 0%。

换句话说，agentic RL 中的熵坍塌不只是“性能饱和”的问题——它会直接导致策略坍塌到一条低质量的捷径上，丧失解决复杂问题的能力。

5.3 对策一：熵奖励项

最直接的应对手段是在优化目标中加入显式的熵奖励项。Reasoning with Exploration: An Entropy Perspective 提出了一个最小化的修改方案——只需一行代码：用熵项增强优势函数。

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_011.png)

这个修改的效果不是“鼓励不确定性”（像传统的 maximum-entropy RL 那样），而是鼓励更长、更深的推理链。

尤其对于Agentic RL来说，高熵区域与三类探索性推理行为正相关：

决定或连接逻辑步骤的关键 token；

反思性行为（自我验证和修正）；

基座模型中欠探索的罕见行为。

在 agentic 场景中，这意味着探索性 tool call 的熵天然较高，熵奖励项能给予这些步骤正向激励，从而对抗前述的“探索步骤低奖励 → 停止探索 → 行为退化”的恶性循环。

这也与标准 PPO 中的 entropy bonus 一致（但传统 entropy bonus 是作为一个正则项加在损失函数中的）：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_012.png)

## 06 PRM 与好奇心奖励

6.1 长程轨迹中的归因困境

在长程智能体轨迹上存在一个核心归因问题：虽然可以用各种方式（蒙特卡罗树搜索（MCTS）、Shapley 值、反事实推理等）去估算每一步对最终结果的因果影响，但这些因果归因往往用于数学等单轮推理场景，很难用于长程智能体轨迹上。

具体来说，MCTS 需要大量的采样，对于我们来说过于昂贵。Shapley 值用于 LLM 奖励归因上时假定每个步骤/token 都是博弈中的一名独立玩家，这个假设在我们的场景下显然是错误的。

单独地移除其中任意一个步骤不仅严重影响后续的步骤的分布，而且解决方案本身也是步骤，最终奖励几乎只与解决问题的步骤直接相关。

反事实推理会受到模型天然的自我纠错机制影响，错误地低估步骤的价值。

6.2 探索的信息价值

关键洞察：在长程轨迹中，有很重要的一部分是智能体的探索。这些探索很多时候是“无结果”的——或者说结果并没有直接贡献给最终的奖励。

但探索依然必要。没有那些探索，就无法消除不确定性，也就无法做出正确的决策。

第 5 章从熵的角度论证了探索的必要性（不探索 → 熵坍塌 → 行为退化）。本章从奖励建模的角度提出：我们应该直接奖励探索行为的信息价值。

已有的 PRM 思路（无论是 learned PRM 还是 heuristic PRM）大多聚焦于“某一步对最终结果的因果贡献”，但忽略了探索步骤的信息价值——这一步虽然没有直接贡献于最终奖励，但它减少了后续决策的不确定性，这种贡献不应该被忽略。

6.3 好奇心奖励的原始构想：事后信息增益

基于上述想法，我们应该用信息论的工具去衡量不确定性，并给予一个”不确定性消除”上的奖励——可以称之为好奇心奖励（Curiosity Reward）。

这个不确定性并不存在于整个轨迹的上下文中，而存在于没有被探索到的剩余信息中。

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_013.png)

这个定义看起来很合理：探索带来的不确定性降低越多，奖励越大。但仔细推敲，它有几个严重的缺陷。

6.4 原始方案的缺陷

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_014.png)

换句话说，这个公式以结果论英雄：它奖励的是”探索恰好有了发现”这个运气结果，而非”做出合理的探索决策”这个行为。

随着训练推进，策略会逐渐放弃那些”事前合理但可能无果”的探索——而这些探索恰恰是解决困难问题所必需的。行为退化由此加速。

反思。上述缺陷指向同一个根源：好奇心奖励应该奖励探索的意图，而非探索的运气结果。

一个动作是否值得探索，应该在执行前就能被评估——取决于它在期望上能带来多少信息。

6.5 关键转折：从”事后熵减”到”预期信息增益”

6.6 理论支撑：内在动机、认知不确定性与世界模型

预期信息增益不是凭空发明——它在理论上扎根于内在动机（Intrinsic Motivation）的经典框架，特别是 Schmidhuber 关于”学习进展（learning progress）”的论述：智能体应该被它”学到了多少新东西”所驱动，而非被”观测结果有多新奇”所驱动。

在这个框架下，有两个核心区分直接影响了我们的设计：认知不确定性 vs. 偶然不确定性。

认知不确定性（Epistemic Uncertainty）：来自”模型知道得不够”——可通过更多数据来降低。例如：没见过这个代码仓库，不确定哪些文件是关键文件。

偶然不确定性（Aleatoric Uncertainty）：来自”世界本身就随机”——无法通过任何数据量来消除。例如：测试通过率本身是概率性的，每次采样有噪声。

好奇心奖励应只响应认知不确定性的降低——即“我学到了东西”——而对偶然不确定性的波动无动于衷。

这天然赋予了好奇心抗噪声的属性：测试通过率的随机抖动不会产生持续的好奇心奖励，因为它们无法被任何模型系统性地减小。

Ensemble 方案的代价。计算认知不确定性的经典方法是维护多个模型的 ensemble，用预测方差作为认知不确定性的估计。

但在 LLM 场景中，维护多个完整模型的 ensemble 是不可承受的——存储开销和推理延迟都不允许。我们需要利用 LLM 本身的特性，设计更轻量的替代方案。

下文 6.7 节的两个组件正是基于这一约束：它们都是轻量附加模块，复用主干网络的隐藏表示，边际成本极低。

6.7 最终方案：两个互补的好奇心组件

综合以上分析——预期信息增益的理论框架、认知不确定性的抗噪优势、LLM 场景对轻量化的要求——我们将好奇心奖励落地为两个互补组件。

两者都度量“学到了什么”，但分别从任务理解和环境建模两个角度切入。

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_018.png)

组件 2：世界模型的学习进步（参考模型作为基线，交叉熵替代损失差）

与其从头训练一个独立的世界模型，更轻量的做法是复用 LLM RL 中已有的参考模型 π_ref（见第 8 节）作为世界模型基线。

LLM 天然具备预测下一 token 的能力——包括环境返回的观测文本——因此π_ref 本身就是一个强大的”先验世界模型”。

世界模型的”进步”等价于其预测能力的提升。具体地π_ref在观测 o_t上的交叉熵损失衡量了该观测相对于参考模型先验知识的”新颖程度”：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_019.png)

直觉检验：

探索了一个全新状态（如读取了一个从未见过的代码库）→π_ref预测很差（交叉熵高）→ 高好奇心奖励。

重复探索已知区域 →π_ref预测良好（交叉熵低）→ 奖励自然消失。

白噪声 →π_ref预测差但因为噪声不可学习、交叉熵不会因模型进步而系统性地降低——但由于我们只做单次前向传播而非追踪差值，白噪声会产生恒定的中等交叉熵。这可以通过阈值过滤或对交叉熵做组内归一化来缓解。

完整的单步好奇心奖励：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_020.png)

与 5.3 节熵奖励项的互补关系：

熵奖励项：保持策略本身的多样性（”让行为池里始终有候选动作”）——供给侧

好奇心奖励：直接度量并奖励”学到了什么”（”从候选池里选出真正有用的探索动作”）——需求侧

两者双管齐下：熵奖励确保不会因为行为单一化而丧失探索能力，好奇心奖励确保探索的方向是有信息价值的，而非随机漫步。

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_021.png)

即在策略的原始概率分布上乘以一个由预期信息增益驱动的偏置项，鼓励智能体在探索阶段早期就倾向于选择”在期望上能带来最大学习进步”的动作。

这在模型对任务还一无所知时尤为有用——预期信息增益可以为探索提供明确的方向感，而非完全依赖随机采样来碰运气。

## 07 从隐式 PRM 到 ORM 驱动的 PRM

上一章讨论了如何给探索步骤赋值（好奇心奖励）。但还有一个更根本的问题没有解决：如何给所有步骤——包括非探索步骤——合理地分配最终奖励的贡献？这就是本章要回答的问题。

7.1 核心问题：给定 ORM，如何构造合理的 PRM？

这个问题的困难在于：没有显式的步骤级标注。我们只知道最终结果，但不知道中间哪一步做对了、哪一步做错了。

## 08 对已有方法的反思：概率扭曲与参考模型

8.1 原始思路

最早提出隐式过程奖励模型的论文使用了一个非常有趣的想法：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_040.png)

核心逻辑：如果模型需要经历很强的扭曲才能输出一个更好的结果，那么这个扭曲就是好的，应该放大它；否则就应该抑制它。

换句话说：

扭曲越强 → 对最终结果的贡献度越大

最终结果为正 → 强扭曲分配到较大的正值

最终结果为负 → 强扭曲分配到较小的负值（惩罚）

即步骤奖励与扭曲程度和最终奖励的乘积成正比：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_041.png)

8.2 问题

这个想法有一个明显的缺陷：我们评分时，不是看扭曲了多少，而是看最终采样的是什么选择。

采样正确 → 得正得分

采样错误 → 得负得分

考虑一个场景：假设参考模型和 RL 模型在某个位置对两个选择（A 正确、B 错误）的概率都是 50%。

此时：

![image](../assets/031_一文讲透长程Agentic_RL中的奖励方法/img_042.png)

更一般地，当参考模型本身并非最优策略时（在复杂 agentic 任务中几乎必然如此），用与参考模型的差异来衡量步骤重要性本身就是有偏的。

参考模型在一个步骤上的高概率选择，可能恰好是需要被纠正的错误选择。

8.3 后续发展

有意思的是，那篇论文在最后甚至把参考模型从理论推导中约掉了，并宣称在 LLaMA 8B 模型上去掉参考模型并不影响最终效果。

但不知道为什么，他们之后的工作 PRIME 并没有复用这个有趣的结论。

## 09 最终的优化目标

综合以上分析，完整的优化目标整合了三条线索：

结果导向的奖励：通过 ORM/价值模型分解最终奖励到各步骤

过程导向的探索奖励：通过好奇心奖励（信息增益）直接激励探索

熵管理：通过熵奖励项对抗策略熵的自然衰减
